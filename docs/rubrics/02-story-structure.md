# ストーリー構成 & 情報設計（配点: 15点）

## 評価の目的
LP全体の情報の流れが、読み手の心理プロセスに沿って設計されているかを評価する。優れたLPは「なぜ今これが必要か」を論理的かつ感情的に積み上げ、CTAへの自然な導線を作る。

---

## スコア段階定義

### 13-15点（優秀）
- PASONA法、AIDCA法、PAS法など実績ある構成パターンに沿っている
- 読み手の心理フロー（問題認識→共感→解決策提示→証拠→行動喚起）が自然
- 各セクションの役割が明確で、重複や冗長がない
- セクション間の遷移がスムーズで「なぜ次にこの情報？」が直感的に分かる
- 離脱ポイント（外部リンク、不要なナビゲーション）が排除されている
- スクロールの深さに応じて説得力が段階的に増す設計

### 9-12点（合格）
- 基本的な構成パターンに沿っているが、一部の順序が不自然
- セクション数は適切だが、1〜2セクションの役割が曖昧
- 離脱ポイントが一部存在する（フッターのリンク過多など）
- ストーリーの流れはあるが、感情的な起伏が弱い

### 5-8点（改善必要）
- セクションの順序に明確な意図が感じられない
- 同じ内容が複数箇所で繰り返されている
- 重要な要素（問題提起、証拠）が欠落している
- 外部リンクや不要なナビゲーションが多く離脱を誘発している
- LP全体が長すぎる or 短すぎる（情報の過不足）

### 0-4点（致命的）
- ストーリーがなく、情報が羅列されているだけ
- セクション間に論理的つながりがない
- LP内に複数のゴール（目的）が混在している
- ユーザーが「なぜこれを読み続けるべきか」の動機が途切れる構成

---

## チェックリスト

- [ ] LP全体が1つの明確なゴール（1アクション）に収束しているか
- [ ] 冒頭で**問題提起 or 共感**があるか（読者の「自分ごと化」）
- [ ] 問題提起の後に**解決策**が提示されているか
- [ ] 解決策の後に**証拠・裏付け**（実績、事例、データ）があるか
- [ ] 証拠の後に**行動喚起**（CTA）があるか
- [ ] 各セクションに明確な見出しがあり、スキャンリーディング可能か
- [ ] 不要な外部リンク・ナビゲーションが最小限か
- [ ] FAQ・反論処理セクションがCTA付近に配置されているか

## 代表的な構成パターン

### PASONA法（問題解決型LP向け）
1. **P**roblem（問題提起）：読者が抱える悩みを具体的に描写
2. **A**ffinity（親近感）：「私も同じでした」と共感
3. **S**olution（解決策）：サービス/商品を提示
4. **O**ffer（提案）：具体的なプラン・価格・条件
5. **N**arrowing（絞り込み）：限定性・緊急性で対象を絞る
6. **A**ction（行動）：CTA

### AIDCA法（高単価商材向け）
1. **A**ttention（注意喚起）：衝撃的な事実やデータ
2. **I**nterest（興味）：ベネフィットの提示
3. **D**esire（欲求）：事例・ストーリーで「欲しい」を喚起
4. **C**onviction（確信）：証拠・保証で不安を解消
5. **A**ction（行動）：CTA

### PAS法（シンプルなLP向け）
1. **P**roblem（問題）
2. **A**gitation（煽り・深掘り）
3. **S**olution（解決策 + CTA）

## Good/Bad パターン

### Good例
- 問題提起（こんな悩みありませんか？）→ 共感（実は多くの企業が…）→ 解決策（当社のサービスなら）→ 実績（導入事例3社）→ FAQ → CTA
  → 読者の心理に沿った自然な流れ

### Bad例
- 会社概要 → 機能一覧 → 価格表 → 問い合わせフォーム
  → 売り手目線の情報羅列。読者の「なぜ必要か」が置き去り
- ファーストビュー → CTA → 機能説明 → CTA → 事例 → CTA → FAQ → CTA
  → CTAの乱用で押し売り感が強く、逆効果

## 業界ベンチマーク
- 1LP=1ゴールの原則に従っているLPは、複数ゴールのLPに比べてCVRが266%高い
- PASONA/AIDCA構成のLPは、構成なしのLPに比べてスクロール完了率が40%以上高い
- 最適なセクション数は5〜8セクション（多すぎても少なすぎてもCVR低下）
